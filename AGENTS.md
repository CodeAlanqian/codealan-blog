
## 本仓库对 AI 助手的要求（根目录）

- 在修改任何代码或文档前，请先阅读 `workflows.md`，按其中约定的流程执行：  
  - 理解需求 → 修改代码/模板/脚本 → 运行 `./build.sh` → 更新文档 → `git add` + `git commit` + `git push origin master`（除非用户特别要求不要提交）。  
- 遵循现有代码风格，不做与当前需求无关的重构。  
- 修改与博客展示相关的内容时，优先调整 Hugo 模板和静态资源，不直接改动 `public/` 下的生成产物。  
- 涉及后端服务时：  
  - 只修改仓库中的脚本（如 `blog_server.py`、`start_backend.sh` 等）；  
  - 对系统级文件（如 `/etc/nginx/...`）只给出配置片段和说明，不直接写入。  
- 运行命令时优先使用现有脚本（如 `./build.sh`、`./start_backend.sh`），避免引入新的复杂工具链。  
- 文档更新请保持中英文风格统一：  
  - 面向读者的概览放在 `README.md`；  
  - 细节实现与运维记录放在 `agentdone.md`；  
  - 工作流与协作约定放在 `workflows.md`。  
- 禁止运行`rm -rf /`等危险命令，避免破坏仓库结构和数据。
- 可以允许有`sudo`权限的命令，但必须确保不会对系统产生不可逆影响。
- 更新学术页时，同时更新中英文版本，保持内容一致。
